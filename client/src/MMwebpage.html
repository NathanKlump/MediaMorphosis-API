MediaMorphosis-API

About

MediaMorphosis API, written in Rust, leverages FFmpeg for high-performance media conversion. It effortlessly transforms audio, video, and images into various formats, ensuring quality and speed. Ideal for developers needing a reliable, swift media converter.

Overview

FFMPEG is a linux command line utility that can converts media formats into others. Our server is created in order to receive an input stream of a media file and a desired file type, feed it into the server and convert the file. Then, it is outputted as a file that contains a stream of data back to the client. We created this server using the programming language Rust and the front end was done with react. Here is the webpage link: https://media-morphosis-api.vercel.app/

Implementation

Our website can be used for media conversion for many use cases. An example of one is converting a video format for mobile device compatibility for video formats that might not be supprted. Another example is content creation for multimedia hobbyists.

Setup Details

To get MediaMorphosis-API up and running on a VM, Rust programming language and FFmpeg have to be installed. Then, clone the project repository and adjust some parts of the configuration files. Specifically, the input/output directories and server ports. Next, configure the React server to connect with the API server. To execute, the user can use the 'start_MM_service.sh script'. The user can also configure the automated log backups as they want.

Group Ownerships

Two user groups were created within the GCP VM: 'Admin' and 'Peasants'. The 'Admin' group has access to logs and system activities, while 'Peasants' are restricted from accessing logs to maintain data integrity.

Restricted Resources

The restricted resource was associated with the video size, and the success rate of the logging has been a bit of an issue due to the middleware. The middleware reads the header bits, which might have a chance of failing, although that hasn't happened yet.

WRITE WHERE TO FIND IT: Script Details

  • • log_network_traffic.sh: Automation of network trafficinf and bandwith usage of the API.
  • • log_ststem_resources.sh: Automation of system resource tracking (ex: CPU and Memory)
  • • log_ststem_resources.sh: This script automates the start of the Rust application and also contains a redirection for the output of to a log file.

WRITE WHERE TO FIND IT: Logging Specifics

Two logging scripts are used:

  • • log_network_traffic.sh: A script that is employed to log the network traffic hitting the API, facilitating both troubleshooting network performance issues and also tracking the utilization of the network bandwidth over time.
  • • log_ststem_resources.sh: This script is very helpful with trying to figure out why the system isn't performing well. Plus, it's always tracking how our system resources are used over time, giving us some really useful insights into their patterns and how they fluctuate in performance.

Backup Script

The backup script we created allows for backup file to be created every 24 hours at 2 am. The file is then archived and zipped. It is located in the following pwd: /home/server/local/CSI3660ProjectBackup. Note: 'CSI3660ProjectBackup' is a directory.
